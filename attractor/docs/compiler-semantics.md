# Compiler Semantics

This pack lowers a static DOT graph into one Gas City `graph.v2` formula.

## Formula Shape

Generated formulas use:

```toml
version = 2
contract = "graph.v2"
```

Every executable DOT node becomes one formula step. The compiler also emits one
hidden body scope step:

```toml
[[steps]]
id = "body"
metadata = { "gc.kind" = "scope", "gc.scope_name" = "attractor", "gc.scope_role" = "body" }
```

The body step depends on every executable step. Each executable step includes:

```toml
metadata = {
  "gc.scope_ref" = "body",
  "gc.scope_role" = "member",
  "gc.on_fail" = "abort_scope"
}
```

Gas City graph controls use that scope metadata to make failures fail-fast:
once one scoped member fails, remaining open scoped work is skipped and the
body scope closes as failed.

## Dependency Lowering

Structural nodes are removed before formula generation. Dependencies are
computed by walking backward from each executable node until the nearest
upstream executable nodes are found.

This means plain static DOT fan-out and fan-in are enough:

```dot
Build -> TestLinux
Build -> TestMac
TestLinux -> Summarize
TestMac -> Summarize
```

`TestLinux` and `TestMac` both need `Build`; `Summarize` needs both test steps.

## Retries

`max_retries=N` on a node compiles to:

```toml
[steps.retry]
max_attempts = N + 1
on_exhausted = "hard_fail"
```

`default_max_retry` and `default_max_retries` on the graph apply to executable
nodes that do not set `max_retries`. A node-level `max_retries=0` disables
retry for that step.

## Tool Nodes

Tool nodes do not execute in the compiler or controller. They become worker
instructions containing the command:

```dot
Verify [shape=parallelogram, tool_command="make test"]
```

The assigned backend worker is responsible for running the command in its
workspace and closing the bead with the standard Attractor close contract.

## Run Targets

`gc_run_target` on an executable node compiles to `metadata["gc.run_target"]`.
It may contain formula variables:

```dot
Build [shape=box, prompt="build", gc_run_target="{{worker_target}}"]
```

The compiler never writes `assignee`; backend worker selection remains a Gas
City dispatch concern.

## Provenance

Generated TOML includes deterministic comments:

```toml
# Generated by gascity-packs attractor 0.1.0
# source = "simple.dot"
# source_sha256 = "..."
```

The compiler does not include timestamps, users, hostnames, current working
directories, or absolute source paths.

Every generated step includes source metadata:

```toml
"attractor.node_id" = "Build"
"attractor.step_id" = "build"
"attractor.node_shape" = "box"
```

## Restart Model

The generated formula stores orchestration state in Gas City work state:
beads, molecule/formula state, dependencies, metadata, and events. Sessions and
workers can restart mid-graph because the authoritative state is not held in
the compiler process. On resume, backend workers and graph controls inspect
the existing work graph and continue from open, unblocked beads.
